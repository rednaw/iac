## How to Contribute

Typical flow:

1. Fork the repo
2. Create a branch
3. Make your changes
4. Run checks if applicable
5. Open a pull request and explain what you changed and why

Small PRs are easier to review and merge.

## Reporting Issues

When opening an issue, it helps to include:

* What you expected to happen
* What actually happened
* How to reproduce it (if possible)
* Relevant logs or snippets
